# 027 - Integration Matrix

> Generated by [027-integration-matrix.gen.py](027-integration-matrix.gen.py). Do not edit by hand вҖ” edit the generator's `EDGES` map and re-run.

Companion artifact for [028-unified-addon-syntax.md](028-unified-addon-syntax.md).
Axes are the **entity names** of every `web-app-*` and `web-svc-*` role.
A cell marks whether the **row** role ships an addon/plugin that wires in the **column** role.

## Legend

| Symbol | Meaning |
|---|---|
| вҳ‘пёҸ | Already wired via an infinito-native service flag in the row role's `meta/services.yml` (group-gated). Links to that declaration. |
| вң… | A verified upstream appвҶ”app plugin exists but is not yet declared as an addon. Links to the plugin page. This is the backlog. |
| рҹӘҷ | Integration exists but is gated behind a commercial / paid tier. |
| вқҢ | No known integration to wire these two. |
| вҖ” | Same role (diagonal). |

Notes:
- The matrix is **directional**: the row hosts the plugin/flag. Bidirectional pairs (e.g. `nextcloud`вҶ”`openproject`) carry a symbol in both cells, each linking to that side.
- вҳ‘пёҸ edges are derived automatically by scanning every role's `meta/services.yml` for integration service keys (`sso`вҶ’keycloak, `matomo`вҶ’matomo, `prometheus`вҶ’prometheus, `email`вҶ’mailu, `dashboard`, `css`, `logout`, `cdn`, `coturn`, `collabora`, `onlyoffice`, `libretranslate`). `ldap`/`redis`/`mariadb` map to `svc-db-*` roles that are off these axes and are not shown.
- `вҶ’ keycloak` вҳ‘пёҸ cells are the central `sso` service (requirement 021); a вң…/рҹӘҷ on `вҶ’ keycloak` instead means a role-local OIDC/SAML addon path beyond the central service.
- Native ActivityPub federation between fediverse roles (`mastodon`, `peertube`, `pixelfed`, `funkwhale`, `mobilizon`, `bookwyrm`, `socialhome`) needs **no plugin** and is therefore not a вң… edge unless an installable connector exists.
